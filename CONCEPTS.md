# Concepts

Shared domain vocabulary for this project — entities, named processes, and status concepts with project-specific meaning. Seeded with core domain vocabulary, then accretes as ce-compound and ce-compound-refresh process learnings; direct edits are fine. Glossary only, not a spec or catch-all.

## Web design system

### Vela
The project's single web design system and brand for all LMS web surfaces — one palette, one component library, one source of truth — anchored on orange as the sole dominant brand color.

### Register
A tuning of the one Vela design system to a surface's job, dialed by "temperature" rather than by forking the system. There are exactly two — the Learner register and the Admin register — which together form the "one spine, two registers" model: one design spine, two temperatures.
*Avoid:* temperature (as a standalone noun)

### Learner register
The warm, motivating Vela register for surfaces aimed at the learner — brighter orange, a wider warm range, and progress/rank/runway motifs that build momentum. Applied to the learner home and the course-canvas surface.

### Admin register
The calm, data-forward Vela register for surfaces aimed at L&D operators — neutral surfaces, a single deep-orange accent, and color used only to carry meaning (on-track / warning / at-risk) so the eye jumps to the rows that need attention. Also called the L&D register.

## Relationships

Vela owns both registers; every web surface renders in exactly one register (Learner or Admin). The registers share Vela's tokens, primitives, and brand — they differ only in temperature, never in spine.
